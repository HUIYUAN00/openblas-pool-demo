## 六、ASAN地址消毒测试结果

### 6.1 ASAN测试概述

AddressSanitizer (ASAN) 是GCC/Clang内置的内存错误检测工具，可检测以下问题：
- 堆缓冲区溢出 (heap-buffer-overflow)
- 栈缓冲区溢出 (stack-buffer-overflow)
- 全局缓冲区溢出 (global-buffer-overflow)
- 使用已释放内存 (use-after-free)
- 双重释放 (double-free)
- 内存泄漏 (memory-leak)

### 6.2 ASAN编译与执行

**编译命令：**
```bash
gcc -g -O1 -Wall -pthread -fsanitize=address -fno-omit-frame-pointer -o test_asan test_full.c pool.c
```

**执行命令：**
```bash
./test_asan
```

|-------|------|------|
| 堆缓冲区溢出检测 | 未发现 | 所有内存访问在合法范围内 |
| 使用已释放内存 | 未发现 | 内存释放后无非法访问 |
| 双重释放检测 | 未发现 | 所有free调用正确 |
| 内存泄漏检测 | 未发现 | 所有分配的内存正确释放 |
